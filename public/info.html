Mystic Tarots

About the project

Hiya!

Thanks for wanting to learn more about this project, in short, it is an assignment for my Master's in Digital Journalism. The course is split between the journalism and computing departments: on the computing side, we learn the basics of HTML, CSS and JavaScript and more. I really wanted to focus on the more technical aspects for this assignment, so I thought: "Let's create an app!"

Where I am by no means a tarot-expert, I find a lot of reassurance in being guided by the cards. While many may not believe in the cards, I know I am not the only one. As I looked to create an app, my mind eventually set on creating a tarot-reading one since I thought it would be useful as an extension of an online magazine or even a news site to drive audiences (kind of like how the games on The New York Times drives subscriptions). Additionally, it gave me the opportunity to not only experiment with the coding part, but also with design: I designed all of the tarot cards as PNG images in Procreate, looking for inspiration from others!

In terms of the coding aspect, I based a lot of my code on the templates shared in the lectures on node, APIs and the SQL database on phpMyAdmin. After some initial struggles, I managed to get it all working locally, but alas, I soon discovered that was just the tip of the iceberg: I had to deploy it! And, in order to deploy it, I could not work with a local API. That is when – as the saying goes – the thickets part of the forest of my adventure began.

First thing first, with zero knowledge of any of this world, I did what many do, I searched the Internet. After a combination of exploring blog sites and asking ChatGPT, I understood the general gist – I had to recreate a database elsewhere and find a server that would support node. I ended up using Railway which offered both a database and the deployment (this also made the paying fee cheaper!)

I initially created a MySQL database in Railway, copying the table format of the phpMyAdmin one. Then, I successfully updated the code to link to that table instead. Following this, I uploaded my files on GitHub so that I could deploy them on Railway. You think that would do the trick but it did not quite do it. There were several issues including not loading the html page that I wanted to be the landing page and the images were no longer showing from my SQL.

I soon realised the deployment had created the API, so, I had to update the code to no longer link to a local one (which it was doing at that point) but rather to the new API. Success! But, I still had the issue with the page. The problem? Rather, problems. There were two: the node code and the file structure. My code was telling the server to respond with a textual message and not with an HTML page. I updated the code to the page I wanted to show, but I would get a 502 Bad Gateway message. This was because my files were all a mess – they were not in the right hierarchy that Railway was expecting.

I moved the files on GitHub to store the "static" ones in the public folder (moving also the PNG images that were in that folder to a sub-folder, and updated the code to not break the path in the node). However, I was unable to redeploy it all successfully. It kept crashing. So, I decided to start from scratch. Rather than updating directly in GitHub, I updated my files locally with all the information and steps I had learned about and then I pushed the updated files to GitHub, anew, via the terminal. AND I MANAGED!

For the "final" steps, I cleaned up the pages, adding as best I could alt-text and making them more mobile responsive (yes, some improvement still pending on these). I learned my lesson and stuck to making the tweaks locally, and later pushing them to GitHub. Next challenge? Make it actually mobile responsice and try and personalise the domain. Stay tuned!

Anyway, enough about the making of the project, can I invite you to test it?

This is a project for my Master's where I am trying to learn the basic ins and outs of html, css, javascript and more, all while experimenting with designs and creativity where I can.
